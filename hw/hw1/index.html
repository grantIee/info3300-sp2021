Problem 3 : Type Coercion

Type coercion is when javascript tries to make the effort to resolve possible compilation errors. Javascript does this through making assumptions about certain objects when they interact with each other.
For example, when doing something like: [[0]] + 1, one might expect this to return an error. In python, because you are adding a list to an int, it is natural for one to expect the compiler to return an error. However, in JS, through type coercion, the browser makes the assumption that the object can be cast to a string and then added to another number. This leads to a confounding result of "01".
Another example, would be the following: null + 2, one might expect this to return an error. In python, because you are adding an object of Nonetype to an int, there will be no supported operator capable of accomplishing this. JS, however, assumes that the null is 0, and adds the two to return the strange result of 2.
